## Lessons Learned

### What Worked

1. **Core profiles are sufficient for routine tasks**
   - Haiku commanders (King, Zhukov, etc.) created quality visuals using 40-line profiles
   - No need for full historical context when task is well-defined
   - **Lesson:** Visual creation tasks can use compact profiles confidently

2. **User enhancement adds significant value**
   - All 5 initial SVG assets were enhanced by user
   - Enhancements: military styling, animations, precise metrics, card layouts
   - **Lesson:** Ship working prototypes, iterate with user feedback

3. **Model tiering works for cost optimization**
   - Haiku handled all visual tasks successfully
   - Sonnet reserved for complex documentation synthesis
   - **Lesson:** Match model to task complexity, not perceived importance

4. **Task-based coordination scales without nesting**
   - Marshall coordinated 8 fronts via TaskCreate/TaskUpdate
   - No nested agent spawning (avoids hierarchy problems)
   - **Lesson:** Coordination != recursive spawning, use task tracking

### What Didn't Work

1. **Agent spawning from teammate context**
   - Marshall (as teammate agent) cannot spawn sub-agents
   - Attempted to use Task tool for spawning, recognized architectural constraint
   - **Lesson:** Coordinators demonstrate structure, don't recursively spawn

2. **README/docs not completed in single session**
   - Tasks #7-8 coordination complete but execution incomplete
   - Complex restructuring requires more context than visual creation
   - **Lesson:** Documentation synthesis is genuinely more complex than visual design
